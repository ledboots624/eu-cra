# EU CRA対策 最終レポート
## SEMI規格準拠企業向け戦略立案

**作成日**: 2026年3月17日
**対象読者**: 経営層・開発責任者・法務担当

---

## 1. エグゼクティブサマリー

EU サイバーレジリエンス法（CRA）は **2026年9月11日に最初の法的義務（脆弱性報告）が発生し、2027年12月11日に全面施行**される。SEMI規格（E187/E188等）への準拠は「出発点」にすぎず、CRA固有の5大ギャップ（脆弱性報告体制・機械可読SBOM・DoC/CEマーキング・5年間サポート義務・EU AR設置）は完全な新規構築が必要である。準拠に要する5年間総費用は基準シナリオで **5,000〜12,000万円**、未対応時の最大罰則は **1,500万ユーロ（約24億円）またはEU市場からの製品排除**であり、経営判断の早期確定が不可欠である。

### 2.2 調査目的

SEMI規格への既存準拠資産を最大限に再利用しながら、最低限のコストでCRA準拠を達成するための以下の成果物を策定する：

- CRA要件×SEMI規格マッピング表
- 3領域別 Must/Want 対応チェックリスト（製品設計・脆弱性管理・ドキュメント適合宣言）
- 段階的導入ロードマップ（フェーズ別・優先順位付き・工数概算付き）
- コスト削減パターン分析（関連規格の重複活用）
- 未対応時のリスクシナリオ整理

### 2.3 調査アプローチ

全6ステップの段階的調査（規制スコープ確認→SEMI×CRAマッピング→技術戦略→責任主体分析→批判的検証→統合）を経て本報告書を作成した。各ステップでの知見の修正・補強を統合し、矛盾を解消した最終版である。

## 3. 主要な発見事項（重要度順）

### 🔴 最優先（即時対応必要）

#### 発見①：2026年9月が最初の法的レッドライン

全面施行（2027年12月）より**1年3ヶ月早く**、Article 14（脆弱性報告義務）が2026年9月11日に適用開始される。この時点までにPSIRT設立・24時間報告体制・ENISA SRP登録が完了していない場合、直ちに法的リスクが発生する。残り期間は約6ヶ月（2026年3月時点）であり、Phase 0（法的基盤確立）を即時開始しなければ間に合わない。

#### 発見②：Phase 0の3タスクがロードマップ全体のゲート条件

以下の3タスクが未確定のまま技術対応を進めると、前提変更による手戻りリスクが極めて高い：

1. **商流・責任主体の確定** — 商社・代理店経由の販売がある場合、輸入者・販売者にも固有のCRA義務（Article 16/17）が発生する。責任分担の空白は、インシデント発生時に製造者責任の範囲拡大につながる。
2. **Annex III製品分類の確定** — 分類（Default / Class I / Class II）によって適合性評価手続き・認証コストが根本的に変わる（後述）。
3. **EU Authorised Representative（AR）の選定・契約** — Article 18によりEU域外製造者のAR設置は「shall」（法的義務）。AR不在ではCEマーキングが法的に不可。

#### 発見③：半導体装置制御ソフトウェアはClass I該当の蓋然性が高い

2025年12月21日施行の実施規則（EU）2025/2392により、Annex III Part IにIACSソフトウェア（SRC-004）が明示的に列挙された。半導体装置の制御ソフトウェアはPLC/DCS/SCADAと同等の機能を持ち、「core functionality」原則でIACS分類に該当する蓋然性が高い。NIS2指令の「essential entities」向けに使用される場合はClass IIへの昇格リスクも存在する。

> **影響**: Default Category前提のコスト試算は過小推計。基準シナリオはClass I前提で設計すること。

### 🟠 重要（Phase 0〜1で対応）

#### 発見④：SEMI準拠はCRAの「証拠」にはならない

SEMI SHOULDとCRA shallは**法的義務レベルが根本的に異なる**。SEMIのSHOULD条項で対応した項目をCRA技術文書に「対応済み」として安易に記載すると、EU市場監視当局の審査で不適合と指摘されるリスクがある。再利用可能範囲はP0-3（SEMI準拠資産の実態棚卸し監査）の結果に完全に依存する。

**CRA要件×SEMI規格 主要マッピング表**

| CRA要件（Annex I） | SEMI E187/E188 対応状況 | 実務上のギャップ |
|---------------------|------------------------|----------------|
| セキュアバイデザイン | **Partial** — E187にセキュリティ要件の概念あり | CRA「shall」水準の実装証拠が別途必要 |
| デフォルトセキュリティ | **Partial** — E187にデフォルト設定要件あり | 条項別の逐条対照と監査結果が必要 |
| 脆弱性管理プロセス | **Partial** — E188に一部規定あり | CRA固有の24h報告・VDPは完全に新規 |
| SBOM（機械可読形式） | **Gap** — E188にソフトウェアインベントリあり | CycloneDX/SPDX形式への変換が必須 |
| 脆弱性報告（24h/ENISAへ） | **Gap** — SEMIに規定なし | 完全新規構築 |
| DoC・CEマーキング | **Gap** — SEMIに規定なし | 完全新規構築 |
| 5年間セキュリティサポート義務 | **Gap** — SEMIに規定なし | 完全新規構築 |
| EU Authorised Representative | **Gap** — SEMIに規定なし | 完全新規構築 |
| 技術文書（Annex V） | **Partial** — 既存文書を流用可能な部分あり | CRA固有フォーマットへの適合が必要 |
| ユーザー向け情報（Annex II） | **Partial** — マニュアル類が基盤になりうる | CRA必須記載事項の追記が必要 |

#### 発見⑤：5年間継続運用コストが総費用の40〜50%を占める

初期構築コストのみで経営判断すると**大幅な過小評価**になる。年間継続運用費（脆弱性監視・パッチ・SBOM更新・インシデント対応等）が初期構築と同等規模である：

| コスト構成 | 基準シナリオ（Class I・SEMI準拠標準的・ISO 27001未保有） |
|-----------|-------------------------------------------------------|
| 初期構築（人件費） | 1,600〜2,900万円（322〜575人日） |
| 初期構築（外部費用） | 400〜1,300万円（AR費用・NB審査・コンサル等） |
| 年間運用（人件費） | 265〜525万円/年（53〜105人日） |
| 年間運用（外部費用） | 80〜200万円/年 |
| **5年間総費用** | **5,000〜12,000万円** |

### 🟡 重要度高（Phase 1〜2で対応）

#### 発見⑥：24時間報告義務の実運用は想定以上に複雑

「24時間以内にENISAに報告」という概念は単純だが、実運用には以下の課題がある：

- **実質対応可能時間**: 24h − 9h（TZ差） − 4h（トリアージ） = **約11時間**
- **タイムゾーン問題**: 欧州の夕方18:00 = 日本の深夜02:00（オンコール体制が必須）
- **3段階報告テンプレートの事前準備**: 24h早期警告・72h詳細・14d最終報告を英語で事前整備
- **ENISA SRPの不確実性**: 2026年3月時点でパイロット段階 → 代替CSIRT経路の並行準備が必須
- **「actively exploited」の判定基準が未確定**: 委任法令で未規定のため、暫定社内基準（CISA KEV掲載/PoC公開/顧客報告）を先行策定

#### 発見⑦：SBOM生成は組込みソフトウェア固有の課題がある

OSSツール（Syft、Trivy等）の多くはLinuxコンテナ環境・標準パッケージマネージャを前提としており、Windows環境・専用ビルドシステム・クローズドコンパイラの半導体装置制御ソフトウェアでは：

- 手動でのSBOMフィールド補完が必要なケースがある
- ビルドシステムプラグインの自社開発が必要になる可能性
- SEMI E188のソフトウェアインベントリは**出発点として活用可能**だが、CycloneDX/SPDX形式への変換・依存関係の深度拡張・ハッシュ値付与が必須

### 4.5 コスト削減パターン：関連規格の重複活用

#### IEC 62443との重複活用（最推奨）

| 効果 | 状況（2026年3月） |
|------|-----------------|
| CRA調和規格への指定（「準拠推定」効果） | CEN-CENELECが修正作業中（2026年中目標）。**正式指定未完了** |
| 技術文書の証拠補強（指定前でも有効） | 今すぐ着手可能 |
| 実現可能なコスト削減効果 | **10〜30%**（指定前）。指定後はさらに拡大 |

> **注意**: 「30〜50%削減」は楽観バイアスがある。2026年3月時点では「10〜30%」が現実的。

#### ISO 27001との重複活用

| 活用可能領域 | 効果 |
|-------------|------|
| リスク管理プロセスをCRA技術文書に転用 | 文書作成工数削減（保有企業のみ） |
| インシデント管理体制をPSIRTに拡張 | PSIRT構築コストの部分的削減 |
| 内部監査プロセスの共用化 | 継続運用コストの削減 |
| 削減効果 | ISO 27001保有企業で**楽観シナリオへの移行**（約2,000〜3,000万円の差） |

### 4.6 未対応リスクシナリオ（罰則・市場リスク）

#### 罰則の定量整理（経営層向け）

| 違反区分 | 罰則上限 | 換算例（売上高100億円企業） |
|---------|---------|--------------------------|
| 重大違反（Annex I必須要件等） | 1,500万ユーロ **または** 全世界売上高の2.5% | **24億円 または 2.5億円（高い方が適用）** |
| 中程度違反（文書・情報提供） | 1,000万ユーロ または 売上高2% | 16億円 または 2億円 |

> **事業インパクト評価**: 日本の半導体装置関連企業のEU売上比率を考えると、罰金よりも「市場アクセス停止」が事業継続上の最大脅威。欧州規制当局との対立は業界評判にも直結する。

### 6.3 調査の信頼性評価

| 指標 | 評価 | 補足 |
|------|------|------|
| 法令要件の正確性 | **高** | EU官報原文（Regulation (EU) 2024/2847、実施規則(EU) 2025/2392）に基づく |
| SEMI×CRAマッピングの精度 | **中** | 規格構造の比較分析に基づく。条項レベルの精密マッピングは監査前に確定不能 |
| コスト試算の信頼性 | **低** | すべて著者推論。独立ベンチマークデータなし。3シナリオ提示により不確実性は明示 |
| ロードマップの実行可能性 | **中〜高** | フェーズ構造・優先順位・依存関係は論理的に整合。工数は監査結果で精緻化が必要 |

**総合評価**: 本報告書は「**何をすべきか（What）**」と「**いつまでに（When）**」については高い信頼度で結論を提示できている。「**いくらかかるか（How much）**」については3シナリオの枠組みは適切だが定量精度は低い。「**具体的にどうやるか（How）**」は自社製品・商流・SEMI準拠実態の固有条件に依存するため、Phase 0完了後に具体化すべきである。
